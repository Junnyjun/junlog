# 접근 방법

파일에 저장된 정보에 접근하는 방식은 응용 프로그램의 성격과 파일 시스템의 설계에 따라 달라집니다. 운영체제는 크게 두 가지 주요 접근 방식과 이를 보완하는 인덱스 방식을 지원합니다.

**1. 순차 접근 (Sequential Access)**

순차 접근은 파일의 정보를 저장된 순서대로 한 번에 하나씩 처리하는 방식입니다. 가장 고전적이고 단순한 방식이며, 컴파일러나 텍스트 에디터에서 주로 사용됩니다.

* 작동 원리:
  * 읽기 연산은 파일의 현재 위치에서 데이터를 읽고 포인터를 자동으로 다음 위치로 이동시킵니다.
  * 쓰기 연산은 파일의 끝에 데이터를 추가하고 포인터를 새로운 끝 지점으로 이동시킵니다.
* 주요 연산:
  * Read Next: 현재 포인터 위치의 데이터를 읽고 포인터를 증가시킵니다.
  * Write Next: 파일의 끝에 데이터를 기록하고 포인터를 증가시킵니다.
  * Reset: 포인터를 파일의 맨 처음으로 되돌립니다.
* 특징: 카세트테이프와 같은 매체의 작동 방식과 유사하며, 대량의 데이터를 처음부터 끝까지 처리할 때 효율적입니다.

**2. 직접 접근 (Direct Access / Relative Access)**

직접 접근은 파일의 블록이나 레코드를 순서에 상관없이 임의의 순서로 읽거나 쓸 수 있는 방식입니다. 대규모 데이터베이스 시스템에서 특정 레코드를 즉시 찾아야 할 때 필수적입니다.

* 작동 원리:
  * 파일을 고정된 크기의 논리적 레코드의 집합으로 간주합니다.
  * 사용자는 레코드 번호(상대적 블록 번호)를 지정하여 즉시 해당 위치로 접근할 수 있습니다.
* 주요 연산:
  * Read $$ $n$ $$: $$ $n$ $$번째 레코드를 읽습니다.
  * Write $$ $n$ $$: $$ $n$ $$번째 레코드에 데이터를 씁니다.
  * Position to $$ $n$ $$: 파일 포인터를 $$ $n$ $$번째 레코드로 이동시킵니다.
* 특징: 디스크 장치는 물리적으로 직접 접근이 가능하므로, 이를 추상화한 직접 접근 방식은 매우 빠른 데이터 검색을 가능하게 합니다.

**3. 기타 접근 방법 (Other Access Methods)**

직접 접근 방식을 기반으로 하여 더 복잡한 검색 기능을 제공하는 방식입니다.

* 인덱스 접근 (Indexed Access):
  * 원리: 파일의 데이터와 별도로 핵심 키(Key)와 해당 데이터의 포인터를 담은 인덱스(Index) 파일을 유지합니다.
  * 작동: 사용자가 특정 키를 찾으면 시스템은 먼저 인덱스에서 포인터를 찾은 뒤, 실제 데이터 파일의 해당 위치로 직접 접근합니다.
  * 장점: 파일이 매우 클 경우 전체를 뒤지지 않고도 원하는 정보를 매우 빠르게 찾을 수 있습니다.
* 다단계 인덱스:
  * 인덱스 파일 자체가 너무 커지면 인덱스에 대한 인덱스를 다시 만드는 방식으로 계층적 구조를 형성합니다.
